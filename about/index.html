How to use the website

'Everything Colors' is a website I built to manipulate and analyze images. Features include pixel sorting, color histogram, color palette generation, and image blending. All these processes are done client-side with JavaScript. You can find out more about these processes below.

To use the website, follow the instructions on the screen and input an image by dropping it anywhere on the screen when prompted to. Alternatively, you can click on the prompt to add images manually. In the 'BLEND' tab, it requires two images. Once the required(s) have been inputted, you can similarly input more images.

DISCLAIMER: images with a very high resolution tend to take longer to process.

Sorted Image

This feature analyzes each pixel of the image input and sorts them by hue. Initially, each pixel is stored and read as RGBA, during the process, it is converted to HSL. Afterward, the pixels are sorted by hue, the degree on a color wheel which is on a scale from 0 to 360. Pixels with a lower hue degree are placed at the top while higher degrees of hue are near the bottom of the image.

Color histogram in RGB space

The histogram is a representation of the image concerning RGB values and each value's frequency in the image. The x-axis is on a scale from 0 to 255 which represents the color's value for that particular channel. The y-axis measures the frequency of each color value. In the histogram, all three channels (red, blue, and green) are presented. Therefore, a larger bar means it is featured in the image more frequently.

Color palette

The color palette generates a palette of 4 colors based on the image. It uses a median cut algorithm to sort the pixels which is done recursively. A color API is used to supply each color with an approximate name. Additionally, you can click on the color display to copy the hexadecimal representation of the color.

Color spaces

This feature includes 3 graphs.

  • Graph 1: x: Red, y: Blue
  • Graph 2: x: Green, y: Red
  • Graph 3: x: Blue, y: Green

A single channel is ignored in each graph due to the limitations of the 2-dimensional graph. Each axis scales from 0 to 255 representing the value in the particular color channel.

It is important to note that this is not a fully accurate representation. To avoid high wait times, this feature downscales the image before rendering the graphs.

Blend

The blend feature requires two images to function. It compares each pixel of one image with the other. Depending on the blend mode selected, it compares the red, green, and blue value of each pixel and places it in the resulting image. The blend modes are commutative meaning the result would be the same even if the images are put in a different order.

There are 3 blend modes, XOR, AND, and OR. Each one represents the logical operation used to compare the pixels.

The dimensions of the resulting image are equal to the minimum dimensions between the two images. This means the minimum width and the minimum height between the two images are taken. As a result, if the dimensions of the two images vary greatly it could mean that the larger image would be 'cropped'.

Instant Alpha

This feature treats the image input as a grid and uses a Depth-First search algorithm. The slider defines the intensity. Increasing the slider value would lower the needed similarity needed to continue the algorithm. The algorithm compares pixel color values in YUV.

Optical Character Recognition

OCR (Optical Character Recognition) will attempt to convert an image which contains text into a digital format using Tesseract.js.

3D Graph

The 3D graph feature represents image input as a graph using Plotly JavaScript Open Source Graphing Library. The feature offers the choice of two types of graphs which are "scatter" (scatter3d) and "mesh" (mesh3d). The graph represents the RGB values of the pixel (0 to 255) on the x, y, and z-axis. Additionally, the feature downscales image to prevent it from taking a long time to process.
